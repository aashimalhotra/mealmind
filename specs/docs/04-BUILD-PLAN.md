# MealMind — Build Plan

## Phase 1: MVP — "Plan & Prep"

### Sprint 1: Foundation (Week 1-2)

**Backend**
- [ ] FastAPI project scaffold with SQLite
- [ ] Database models: users, household, recipes, meal_plans
- [ ] CRUD endpoints for recipes and profile
- [ ] LiteLLM proxy configuration (Ollama + cloud model)
- [ ] Docker Compose setup for local dev

**Frontend**
- [ ] Vite + React + Tailwind PWA scaffold
- [ ] PWA manifest + service worker (installable)
- [ ] Design system implementation: CSS variables, color tokens, typography scale
- [ ] Bottom navigation component (Plan / Recipes / Profile)
- [ ] Layout shell with routing

### Sprint 2: Dashboard (Week 3)

- [ ] Macro ring component (SVG, animated fill)
- [ ] Dashboard screen: greeting, four macro rings, calorie summary
- [ ] Meal card component with type badges
- [ ] Today's meals section (three meal cards)
- [ ] Week strip component (horizontal scroll, day pills)
- [ ] Prep day card component
- [ ] AI insight card component
- [ ] Person toggle (1500 / 1800 view switching)

### Sprint 3: Meal Plan Generation (Week 4)

- [ ] Plan generation API endpoint (streaming)
- [ ] LLM system prompt for plan generation (preferences, constraints, history)
- [ ] Plan approval/edit flow in UI
- [ ] Weekly plan data structure and storage
- [ ] Dashboard ↔ plan data binding (show real meals from approved plan)
- [ ] Dine-out day handling (badge, no meal card)

### Sprint 4: Recipe Detail (Week 5)

- [ ] Recipe detail screen (full layout)
- [ ] "Prepped on [day]" status badge
- [ ] Serving instructions section (reheat + plate)
- [ ] Ingredient list with 1500/1800 portion toggle
- [ ] Storage & reheating section
- [ ] AI serving tip card
- [ ] "View in prep guide" CTA
- [ ] Tags display (auto-generated by LLM on recipe save)
- [ ] Navigation: dashboard meal card → recipe detail

### Sprint 5: Prep Guide (Week 6-7)

- [ ] Prep guide screen (full layout)
- [ ] LLM prompt for prep step sequencing (interleave multiple recipes)
- [ ] Active step card with large text and timer
- [ ] Timer component: countdown, pause, extend (+1 min)
- [ ] Background timer list (parallel tasks)
- [ ] Step progression: completed → active → upcoming
- [ ] Progress bar (color-coded by dish)
- [ ] Batch overview header (dishes being prepped)
- [ ] Navigation: dashboard "Start prep guide" → prep guide
- [ ] Navigation: recipe detail "View in prep guide" → prep guide

### Sprint 6: Grocery List (Week 8)

- [ ] Grocery list generation API (LLM consolidates ingredients across recipes + both calorie targets)
- [ ] Grocery list screen layout
- [ ] Category grouping (Protein, Produce, Dairy, Grains & Pantry, Spices)
- [ ] Filter tabs: All / Sun prep / Wed prep / Day-of
- [ ] Item check/uncheck with strikethrough
- [ ] Pantry check section (tappable chips for spices/staples)
- [ ] Recipe attribution per item (subtitle: "Sun prep — tandoori chicken")
- [ ] Search bar
- [ ] Share list / Copy as text
- [ ] Navigation: accessible from dashboard and AI chat

### Sprint 7: AI Chat Panel (Week 9)

- [ ] Chat API endpoint (streaming, context-aware)
- [ ] Chat copilot system prompt (current plan, screen context, preferences)
- [ ] Bottom sheet component (drag handle, half/full screen)
- [ ] FAB button (saffron gold, positioned above nav bar)
- [ ] Chat UI: AI bubbles, user bubbles, timestamps
- [ ] Inline recipe card component (within chat suggestions)
- [ ] Action buttons on suggestions ("Add to plan" / "Suggest another")
- [ ] Quick action chips (leftover ideas, swap meal, review nutrition, regen grocery)
- [ ] FAB pulse animation for proactive suggestions
- [ ] Proactive insight on plan load (nutritional gap check)

### Sprint 8: Polish & Deploy (Week 10)

- [ ] End-to-end flow testing: generate plan → review → approve → prep guide → grocery list
- [ ] PWA install testing on iOS and Android
- [ ] Offline support for approved plans and grocery list (service worker caching)
- [ ] Docker Compose finalization
- [ ] Deploy to Proxmox VM
- [ ] Tailscale setup for mobile access
- [ ] Seed initial recipes (tandoori chicken, lentil soup, cumin rice, aloo gobi, keema, palak paneer, etc.)

---

## Phase 2: Nutrition Brain (Week 11-14)

- [ ] Per-meal macro tracking (LLM estimates macros from recipe descriptions)
- [ ] Daily macro progress update on dashboard (rings fill based on completed meals)
- [ ] Weekly nutritional gap analysis endpoint
- [ ] Proactive AI insight: flag low iron, fiber, veggie shortfalls
- [ ] Recipe memory: favorite/dislike toggles persisted and injected into plan generation prompt
- [ ] USDA FoodData Central API integration for macro validation (optional, accuracy layer)
- [ ] Plan history (last 4 weeks) injected into generation prompt to avoid repetition

---

## Phase 3: Enhanced Copilot (Week 15-18)

- [ ] Chat history persistence across sessions
- [ ] Full-screen chat expansion
- [ ] Mid-prep contextual Q&A (substitutions, troubleshooting)
- [ ] "What can I make with leftovers?" intelligence
- [ ] LLM function calling for in-chat actions (swap meals, add grocery items, generate recipes)
- [ ] Prep coaching mode (step-by-step guidance with embedded timers in chat)

---

## Phase 4: Scale & Publish (Future)

- [ ] Multi-user auth (email + password or magic link)
- [ ] Household invite system
- [ ] Template caching layer (pre-generated plan patterns, LLM personalizes)
- [ ] Collaborative filtering from recipe corpus (users who liked X also liked Y)
- [ ] Google Calendar integration (prep day reminders, dine-out blocks)
- [ ] Photo-based meal logging (LLM estimates macros from image)
- [ ] Seasonal ingredient suggestions
- [ ] Multi-cuisine support (Mediterranean, East Asian, Mexican, etc.)
- [ ] Public launch: app store listing, landing page, pricing tiers
